4 · Guardrails for parallel work

So we don't collide:

  1. v1 files don't move. Same filenames, same URLs. I only add, never restructure.
  2. New wireframe screens register via window.XxxScreens.push — your bookmarks stay valid.
  3. Data Model entity blocks are append-only. Phase 1 entities keep their exact shape.
  4. README gets new sections appended under existing numbered headers; existing sections don't renumber.
  5. Design System tokens are frozen. If a new pattern is needed for v2 (e.g. MapView, RouteLine), I document it in an addendum — no retroactive changes.
  6. Build against v1 captions as the source of truth for anything still being thickened. Wireframe captions are the PRD — if a caption says it, the feature is in scope.
