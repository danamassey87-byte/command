Command · Acceptance Tests · v1

Per-phase done criteria

A phase is done when a human (Dana) can click through the test script and every step works. Not "the code compiles." Not "the PR is green." These are the hand-off gates between phases.

How to use — The agent runs each script at the end of its phase and reports PASS / FAIL with evidence (screenshot, log, or DB query). Dana reviews the PASS report before the next phase starts. FAIL means fix-in-place; do not proceed.

Phase 9

Lead sources & pay-at-close attribution

Scope PAC intake, fee %, deal-room deduction, cash-offer routingDuration 3–4 days
Pre-req
  • 7 seed sources inserted: CertiLeads, ClosingLeads, AtClosing, Movoto, Realty.com, LeadDeck, FastCashOffers (see Seed Data §16).
  • Intake credentials configured in /admin/integrations — email forwarders + at least one API poller + cash-offer webhook.
  • Feature flag lead_sources_v1 on.
Test script
  1. Open /lead-sources. Expect all 7 sources listed with correct buyer %, seller %, cost model, attribution window.
  2. Click CertiLeads. Expect detail view with intake config, live totals (leads YTD / closed / fees owed / GCI net), pipeline-fee forecast.
  3. Forward a real CertiLeads lead email to the configured address. Within 2 min expect:
    • Contact created in /crm with tag certileads.
    • lead_attributions row with buyer_pct_locked = 30.00, seller_pct_locked = 25.00, expires_at = null (per-deal).
    • Contact auto-enrolled in "New PAC lead" sequence.
  4. Change CertiLeads buyer % from 30 → 35 in /lead-sources/certileads. Expect the in-flight contact's buyer_pct_locked to stay 30 (fee is locked at arrival). New leads from this point forward should lock at 35.
  5. Open a deal for that contact (under contract). Expect the Deal Room sidebar to show Source · Fee block: CertiLeads · PAC · projected GCI · −$X referral · Dana net.
  6. Mark the deal as closed. Expect:
    • A referral_fees row with status = suggested flipping to confirmed, locked_at set.
    • Close action is gated until the fee is confirmed or explicitly waived with a note.
  7. Override the fee: reduce it by $500 with reason "negotiated partial refund." Expect the audit row on referral_fees with override_reason populated.
  8. Cash-offer routing: submit a lead via the home-value microsite with "cash offer" intent. Expect:
    • cash_offer_routings row created, sla_due_at = now() + 24h.
    • Request pushed to FastCashOffers (mock in dev).
    • If declined: contact auto-enrolls in "Traditional list" sequence.
  9. Analytics: /lead-sources/analytics shows correct ROI per source, $/lead net, 90-day fee forecast equals sum of projected_referral_fee_cents on in-pipeline deals.
Gate: A fee % change must never retroactively modify an already-locked LeadAttribution. Write an explicit regression test.

Non-functional gates (every phase)

  • No TypeScript any in touched files without a // eslint-disable comment explaining why.
  • All new API routes have a Zod schema on input.
  • Row-level security enabled on every new table, with policies that scope to owner_id = auth.uid().
  • No LLM call bypasses lib/ai/bill.ts. Grep for direct anthropic.messages.create imports — none allowed outside that file.
  • Every external API call wrapped with retry + timeout + error logging to integration_errors.
  • Slack callbacks verify signature; payloads older than 5 min are rejected.

When a test fails, fix it before writing new features. Accumulating "known-bad" breaks the value of these gates.
