Indexes + search

pgvector on contacts.voice_notes, content_drafts.caption, interactions.body, all library items. Pinecone is the external RAG index: brain (business doc), compliance (handbooks), voice (Dana's past writing), performance (what worked). Summary cards generated at index time to cut query tokens ~80%.

Sync rules of thumb

  • Transact is source of truth for paperwork. Command reads; never overwrites a Transact field.
  • Lofty is bidirectional for contacts + tags + notes; Lofty-owns saved searches and valuations.
  • Blotato is downstream-only. Command pushes; Blotato owns scheduling.
  • Slack is outbound with inbound callbacks for button actions only — no free-form ingest.
  • Resend + Gmail log to Interactions via webhooks.
  • MLS is manual + AI-researched until API access lands. Keep mls_id nullable.
  • Social platforms — IG/FB/LI via Graph API where possible; TT/X/YT/Threads/Pin/Nextdoor are paste-URL + AI verification until each vendor exists. Never store passwords.
  • AVM providers (Zillow/Redfin/Realtor/Attom) — cache 24h per property_id. Blend only at read time; raw values stored for audit.
  • GMB — Business Profile API scopes business.manage. Posts scheduled ≥10 min out to allow cancel-window.
  • Rank tracking — DataForSEO or SerpAPI. Weekly snapshot only; on-demand for "recheck now" is rate-limited per keyword.
